# Лабораторная работа номер 5, Debug ЛР4 для поиска ошибок (выполнил Малинин Ярослав Юрьевич М8О-102БВ-25)


## ЛР4
Пакет включает в себя реализацию таких классов как:
- Book - обычная книга
- ElectronikBook - электронная книга (наследуется от Book)
- StudentBook - учебник (наследуется от Book)
- BookCollection - коллекция книг
- IndexDict - стуктура данных, содержащая индексы книг BookCollection, хранаящихся по isbn, year, author
- Library - библиотека, хранящая индексы и коллекцию книг

### Структура проекта

- [main.py](src/main.py) - точка входа программы
- [books.py](src/books.py) содержит основные класс, структуры данных, отражающие предметную область библиотеки
- [constants.py](src/constants.py) хранит константы, нужные для реализации вышеперечисленных функций
- [simulation.py](src/simulation.py) реализует функции, симулирующие систему генерации псевдослучайных событий в библиотеки



### Что ожидает программа от пользователя

В программе есть небольшой CLI, с которым пользователь может взаимодействовать и наблюдать за основыными событиями:
- добавление книги
- удаление книги
- очищение билиотеки
- попытка получить несуществующую книгу
- поиск книги по ключу и значению в библотеке


## ЛР5

### Структура проекта

Кроме того что перечислено в структуре проекта ЛР4 были добавлены следующие файлы/папки:

- [mistake_clases.py](src/mistake_clases.py) - пронаследованы классы от базовых из ЛР4, содержащие ошибки
- [mistakes.py](src/mistakes.py) - созданы функции, которые симулируют возникновение ошибок
- [MISTAKES.md](MISTAKES.md) - отчет о проделанной работе

### Что ожидает программа-отчет от пользователя

Подуразмевается, что в файле [MISTAKES.md](MISTAKES.md) указаны ошибки и их место нахождение. Для того, чтобы симулировать ошибку, нужно сделать все, что указано в инструкции. Для того, чтобы посмотреть как программа работает правильно нужно вставить исправленный код в соотв класс и функцию в файле [mistake_clases.py](src/mistake_clases.py) и запустить выполнение программмы заново.
